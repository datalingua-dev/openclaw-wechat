# Fitness Coach Agent (AI 健身教练)

基于 OpenClaw 与企业微信的大模型 AI 健身私教助手。不仅能根据用户身体数据生成和追踪健身计划，更具备强大的**本地沙箱数据隔离**与**主动定时关怀**能力。

## 🌟 核心功能特性

### 1. 🥇 个性化档案与训练计划管理
- **智能档案管理**: 初次交流自动建立私密档案，记录用户的身体指标、活动受限史与过敏史。
- **自适应训练生成**: 针对减脂、增肌、康复等不同目标，动态拆解出详细的周度训练与饮食打卡明细。

### 2. 🔐 多用户级物理数据隔离机制 (Multi-User Data Sandbox)
不同于传统的单文件追加（容易暴露隐私），本框架彻底实现了企业级的数据隔离：
- **独立虚拟空间**: 根据企微 `WeCom_ID` 动态创建物理存储路径。
- **文件隔离墙**: `/users/<wecom_id>/` 目录分别独立存放该用户的 `profile.md`, `memory.md`, `plan.md`，在 Linux 大小写敏感生态下依然能智能防冲突。
- **绝对隐私**: 大模型读取文件时，API Gateway 从物理层强制隔绝，用户 A 永远无法读取到用户 B 的减肥数据。

### 3. ⏰ 大模型自主原生定时通知 (Cron Engine)
内置由 AI 原生驱动的智能调度插件 (`fitness-cron-plugin`)，实现了“像真人教练一样的到点叮嘱”。
- **灵活的周期语法**: 支持 `once`(单次), `daily`(每天), `workdays`(工作日), `weekends`(周末) 甚至指定 `1,3,5` (每周一三五)。
- **持久化任务编排**: 所有日程被写入每位用户的私有 `reminders.json` 文件。网关重启、服务器崩溃重启后亦能准确还原所有用户的提醒配置。
- **场景化下发**: 引擎时间触发后，会构造上下文“唤醒”大模型，大模型阅读用户的最新运动进展后，主动发去拟人化、伴随式的企微消息关怀。

## 📌 架构全景

```text
~/.openclaw/workspace-fitness-coach/
├── users/
│   ├── huanghui/
│   │   ├── plan.md            => 动态生成的训练指引
│   │   ├── memory.md          => 对话偏好与近期状态
│   │   └── reminders.json     => 用户的 Cron 任务队列
│   └── other_user/
```

## 🚀 部署引导

1. 修改 `openclaw.json` 配置：确保应用 ID 为 `fitness-coach` 并启用了 `cron` 权限。
2. 将 `fitness-cron-plugin.js` 编入网关加载列表。
3. Agent 启动后可通过企微扫码绑定。向机器人说“每周一三五早上7点提醒我空腹有氧”，体验全自动排期。
